TypeScript – let & const

Control how variables behave and prevent accidental bugs

Why Use let and const?

Block Scope

Variables declared with let and const only exist inside their block.

Safer Code

Prevent accidental redeclaration and unexpected changes.

Predictable Programs

Using const protects values that should never change.

Code Example


let score: number = 50;

const COUNTRY: string = "India";

score = 75;

console.log("Updated Score:", score);
console.log("Country:", COUNTRY);
  • let score creates a variable whose value can change.
  • const COUNTRY creates a constant that cannot change.
  • score = 75 works because let variables can be reassigned.
  • Trying to change COUNTRY would cause a TypeScript error.

Const Error Example




Constants cannot be reassigned once they are created.





const PI: number = 3.14;

PI = 3.15;
// ❌ TypeScript Error:
// Cannot assign to 'PI' because it is a constant





TypeScript prevents changing constants, helping avoid unexpected bugs.
