Variables in TypeScript

Learn how TypeScript stores and manages data using variables

Understanding Variables

Variables are like labeled drawers in an office. Each drawer has a name and holds a specific type of information. In TypeScript, variables store data such as numbers, text, or boolean values.

Types of Variable Declarations

Type + Value

Declare both the type and the value of a variable.

Type Only

Declare a variable type first, then assign a value later.

Type Inference

TypeScript automatically determines the variable type.

Variable Naming Rules

  • Can include letters, numbers, _ and $
  • Cannot start with a number
  • Cannot contain spaces
  • Cannot use reserved keywords
  • Variable names are case-sensitive

Valid vs Invalid Variable Names


Valid Names

firstName
first_name
num1
$result


Invalid Names

first name
1number
first-name
var

Example: Variables in TypeScript


var name: string = "John";

var score1: number = 50;

var score2: number = 42.5;

var sum = score1 + score2;

console.log("name: " + name);
console.log("first score: " + score1);
console.log("second score: " + score2);
console.log("sum of scores: " + sum);

Strong Typing

TypeScript prevents assigning incorrect types. 


var num: number = "hello"; // Compilation Error

Type Assertion


var str = "1";

var str2: number = <number><any>str;

console.log(typeof(str2));

Variable Scope


var global_num = 12;

class Numbers {

num_val = 13;

static sval = 10;

storeNum(): void {

var local_num = 14;

}

}

console.log(global_num);
console.log(Numbers.sval);

var obj = new Numbers();
console.log(obj.num_val);

Key Takeaways

  • Variables store data in TypeScript programs.
  • They follow specific naming rules.
  • Types help prevent programming errors.
  • Type inference allows TypeScript to detect types automatically.
  • Variable scope determines where variables can be accessed.
